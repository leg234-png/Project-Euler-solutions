# Combinaisons Sélectionnées

## Description du Problème
Il y a exactement dix façons de sélectionner 3 éléments parmi 5 : 123, 124, 125, 134, 135, 145, 234, 235, 245, et 345.

En combinatoire, on utilise la notation C(5,3) = 10.

En général, C(n,r) = n!/(r!(n-r)!) où r ≤ n, n! = n×(n-1)×...×3×2×1, et 0! = 1.

Il n'est pas jusqu'à n = 23 qu'une valeur dépasse un million : C(23,10) = 1144066.

**Objectif** : Combien de valeurs de C(n,r) pour 1 ≤ n ≤ 100 sont supérieures à un million ?

---

## Exemples
### Exemple 1
**Entrée** : n = 5, r = 3
**Sortie** : `10`
**Explication** : C(5,3) = 10.

### Exemple 2
**Entrée** : n = 23, r = 10
**Sortie** : `1144066`
**Explication** : C(23,10) dépasse un million.

---

## Approche de Résolution

### Stratégie
1. Pour chaque n de 1 à 100 :
   - Pour chaque r de 0 à n :
     - Calculer C(n, r) en utilisant BigInteger.
     - Si C(n, r) > 1 000 000, incrémenter le compteur.
2. Retourner le compteur.

### Observations
- On peut utiliser la propriété C(n, r) = C(n, n-r) pour optimiser.
- On doit utiliser BigInteger car les valeurs peuvent être très grandes.
- On peut calculer le coefficient binomial de manière itérative pour éviter les débordements.

### Complexité
- **Temporelle** : O(n² × log n), où n = 100.
- **Spatiale** : O(log n), pour stocker les BigInteger.

---

## Solution en C++

```cpp
#include <iostream>
#include <gmpxx.h>

mpz_class binomial(int n, int k) {
    if (k > n - k) {
        k = n - k;  // Optimisation : C(n, k) = C(n, n-k)
    }
    
    mpz_class result = 1;
    for (int i = 0; i < k; ++i) {
        result = result * (n - i) / (i + 1);
    }
    
    return result;
}

int main() {
    mpz_class MILLION = 1000000;
    int count = 0;
    
    for (int n = 1; n <= 100; ++n) {
        for (int r = 0; r <= n; ++r) {
            if (binomial(n, r) > MILLION) {
                count++;
            }
        }
    }
    
    std::cout << count << std::endl;
    return 0;
}
```
