# Triangles Rectangles Entiers

## Description du Problème
Si p est le périmètre d'un triangle rectangle avec des côtés de longueur entière, {a, b, c}, il y a exactement trois solutions pour p = 120.

{20, 48, 52}, {24, 45, 51}, {30, 40, 50}

**Objectif** : Pour quelle valeur de p ≤ 1000, le nombre de solutions est-il maximisé ?

---

## Exemples
### Exemple 1
**Entrée** : p = 120
**Sortie** : `3`
**Explication** : Il y a 3 triangles rectangles avec un périmètre de 120.

---

## Approche de Résolution

### Stratégie
1. Pour chaque périmètre p de 1 à 1000 :
   - Parcourir toutes les valeurs possibles de a et b.
   - Calculer c = p - a - b.
   - Vérifier si a² + b² = c² et si a ≤ b ≤ c.
   - Compter le nombre de solutions.
2. Retourner le périmètre avec le maximum de solutions.

### Observations
- On peut optimiser en remarquant que a < p/3 (sinon a + b + c > p).
- On peut aussi utiliser des formules pour générer des triplets pythagoriciens, mais pour p ≤ 1000, la recherche exhaustive est suffisante.
- On doit s'assurer que b ≤ c pour éviter de compter deux fois le même triangle.

### Complexité
- **Temporelle** : O(p³), où `p` = 1000.
- **Spatiale** : O(1), car on utilise seulement quelques variables.

---

## Solution en C++

```cpp
#include <iostream>

int countSolutions(int p) {
    int count = 0;
    for (int a = 1; a <= p; ++a) {
        for (int b = a; b <= p; ++b) {
            int c = p - a - b;
            if (b <= c && a * a + b * b == c * c) {
                count++;
            }
        }
    }
    return count;
}

int main() {
    int maxPerimeter = 0;
    int maxTriangles = 0;
    
    for (int p = 1; p <= 1000; ++p) {
        int triangles = countSolutions(p);
        if (triangles > maxTriangles) {
            maxTriangles = triangles;
            maxPerimeter = p;
        }
    }
    
    std::cout << maxPerimeter << std::endl;
    return 0;
}
```
