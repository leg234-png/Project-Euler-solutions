# Triangles rectangles à côtés entiers uniques

## Description du Problème
On dit qu'un ensemble de trois entiers positifs (a, b, c) est un triplet pythagoricien si a² + b² = c².

Par exemple, (3, 4, 5) est un triplet pythagoricien.

Il existe exactement un triplet pythagoricien pour lequel a + b + c = 1000. Trouvez le produit abc.

**Objectif** : Il s'avère que 12 cm est la plus petite longueur de fil qui peut être pliée pour former un triangle rectangle à côtés entiers de manière exactement une seule façon, mais il existe de nombreuses autres longueurs.

Par exemple, si on utilise exactement 120 cm de fil, il existe exactement trois façons différentes de plier le fil pour former un triangle rectangle à côtés entiers :
- 120 cm : (30, 40, 50), (20, 48, 52), (24, 45, 51)

Étant donné que L est la longueur du fil, pour combien de valeurs de L ≤ 1 500 000 peut-on former exactement un triangle rectangle à côtés entiers ?

---

## Exemples
### Exemple 1
**Entrée** : L = 120
**Sortie** : `3`
**Explication** : Il y a 3 façons de former un triangle rectangle avec 120 cm : (30,40,50), (20,48,52), (24,45,51).

---

## Approche de Résolution

### Stratégie
1. Générer tous les triplets pythagoriciens primitifs en utilisant la formule d'Euclide.
2. Pour chaque triplet primitif (a, b, c) avec périmètre p, marquer tous les multiples kp ≤ 1 500 000.
3. Utiliser un tableau pour compter combien de façons chaque périmètre peut être formé.
4. Compter combien de périmètres ont exactement 1 façon.

### Observations
- Théorème des triplets pythagoriciens : Chaque triplet primitif avec a impair et b pair peut être exprimé comme :
  - a = st, b = (s²-t²)/2, c = (s²+t²)/2
  - où s > t > 0 sont des entiers impairs copremiers.
- On génère tous les triplets primitifs et on les met à l'échelle pour trouver tous les triplets.
- On utilise un tableau de bytes pour compter les façons, en saturant à 2 pour optimiser.

### Complexité
- **Temporelle** : O(LIMIT), où LIMIT = 1 500 000.
- **Spatiale** : O(LIMIT), pour stocker le tableau de comptage.

---

## Solution en C++

```cpp
#include <iostream>
#include <vector>
#include <set>
#include <algorithm>

using namespace std;

struct IntTriple {
    int a, b, c;
    IntTriple(int a, int b, int c) : a(a), b(b), c(c) {}
    bool operator<(const IntTriple& other) const {
        if (a != other.a) return a < other.a;
        if (b != other.b) return b < other.b;
        return c < other.c;
    }
};

int gcd(int a, int b) {
    while (b != 0) {
        int temp = b;
        b = a % b;
        a = temp;
    }
    return a;
}

int main() {
    const int LIMIT = 1500000;
    set<IntTriple> triples;
    
    // Générer les triplets primitifs
    for (int s = 3; s * s <= LIMIT; s += 2) {
        for (int t = s - 2; t > 0; t -= 2) {
            if (gcd(s, t) == 1) {
                int a = s * t;
                int b = (s * s - t * t) / 2;
                int c = (s * s + t * t) / 2;
                if (a + b + c <= LIMIT)
                    triples.insert(IntTriple(a, b, c));
            }
        }
    }
    
    vector<int> ways(LIMIT + 1, 0);
    for (const auto& triple : triples) {
        int sum = triple.a + triple.b + triple.c;
        for (int i = sum; i <= LIMIT; i += sum) {
            if (ways[i] < 2)
                ways[i]++;
        }
    }
    
    int count = 0;
    for (int x : ways) {
        if (x == 1)
            count++;
    }
    
    cout << count << endl;
    return 0;
}
```
