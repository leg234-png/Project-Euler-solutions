# Prime Pair Connection

## Description du Problème
Considérons les nombres premiers consécutifs p₁ = 19 et p₂ = 23. On peut vérifier que 1219 est le plus petit nombre tel que les derniers chiffres sont formés par p₁ tout en étant divisible par p₂.

En fait, à l'exception de p₁ = 3 et p₂ = 5, pour chaque paire de nombres premiers consécutifs, p₂ > p₁, il existe des valeurs de n pour lesquelles les derniers chiffres sont formés par p₁ et n est divisible par p₂. Soit S la plus petite de ces valeurs de n.

**Objectif** : Trouver ∑S pour chaque paire de nombres premiers consécutifs avec 5 ≤ p₁ ≤ 1000000.

---

## Exemples
### Exemple 1
**Entrée** : `p₁ = 19, p₂ = 23`
**Sortie** : `1219`
**Explication** : 1219 se termine par 19 et est divisible par 23.

---

## Approche de Résolution

### Stratégie
1. Pour chaque paire (p₁, p₂), trouver le plus petit n tel que n ≡ p₁ mod 10ᵏ et n ≡ 0 mod p₂.
2. Soit k la plus petite puissance de 10 qui dépasse p₁.
3. On cherche n = mk + p₁ tel que n ≡ 0 mod p₂.
4. Cela donne m ≡ -p₁ × k⁻¹ mod p₂.
5. Utiliser l'inverse modulaire pour trouver m.

### Observations
- On peut utiliser l'algorithme d'Euclide étendu pour trouver l'inverse modulaire.
- k⁻¹ existe car p₂ est copremier avec 10.

### Complexité
- **Temporelle** : O(n log p), où n est le nombre de paires de nombres premiers.
- **Spatiale** : O(n) pour stocker les nombres premiers.

---

#include <iostream>
#include <vector>

long long reciprocalMod(long long x, long long m) {
    // Algorithme d'Euclide étendu pour trouver l'inverse modulaire
    long long a = x % m;
    long long b = m;
    long long x0 = 1, x1 = 0;
    
    while (a > 1) {
        long long q = a / b;
        long long temp = b;
        b = a % b;
        a = temp;
        temp = x1;
        x1 = x0 - q * x1;
        x0 = temp;
    }
    if (x0 < 0) x0 += m;
    return x0;
}
