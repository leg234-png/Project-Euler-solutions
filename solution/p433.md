# Étapes dans l'algorithme d'Euclide

## Description du Problème
Soit E(x, y) le nombre d'étapes nécessaires pour calculer le PGCD de x et y avec l'algorithme d'Euclide. Plus formellement, E(x, y) est le plus petit k >= 0 tel que dans l'algorithme d'Euclide appliqué à (x, y), après k étapes, on obtient un couple où l'un des éléments est 0.

Par exemple, E(5, 8) = 3 car (5, 8) -> (8, 5) -> (5, 3) -> (3, 2) -> (2, 1) -> (1, 0).

**Objectif** : Trouver ΣE(x, y) pour 1 <= x, y <= 5×10⁶.

---

## Exemples
### Exemple 1
**Entrée** : `x = 5, y = 8`
**Sortie** : `3`
**Explication** : E(5, 8) = 3 étapes.

---

## Approche de Résolution

### Stratégie
1. Considérer les paires (x, y) où x > y >= 0.
2. Pour une paire (x, y) avec y > 0, une étape de l'algorithme d'Euclide produit une nouvelle paire (y, z) où z = x mod y.
3. Pour trouver toutes les paires (x, y) qui produisent une paire donnée (y, z), calculer toutes les façons possibles de prendre une étape inverse : x = z + ky, où k >= 1 (ou k >= 2 si z = 0).
4. Utiliser une recherche en profondeur d'abord en partant de chaque racine (z, 0) et en faisant des pas inverses jusqu'à ce que x > LIMIT.
5. Pour les autres cas (x = y ou x < y), utiliser les relations :
   - E(x, x) = 1
   - E(x, y) = E(y, x) + 1 si x < y

### Observations
- On explore toutes les paires en utilisant des pas inverses de l'algorithme d'Euclide.
- On commence à partir de chaque racine (z, 0) et on construit récursivement toutes les paires possibles.
- La somme finale est 2s + LIMIT×(LIMIT+1)/2, où s est la somme pour 1 <= y < x <= LIMIT.

### Complexité
- **Temporelle** : O(N²), où N = 5×10⁶ (très long).
- **Spatiale** : O(1), pour la récursion.

---

## Solution en C++

```cpp
#include <iostream>

using namespace std;

const int LIMIT = 5000000;

long long sum = 0;

// Nécessite 0 <= y < x <= LIMIT
void exploreGcds(int x, int y, int steps) {
    if (!(0 <= y && y < x && x <= LIMIT)) {
        return;
    }
    sum += steps;
    
    // Faire des pas inverses : x = z + ky où z = y mod x
    // Si y > 0, alors k >= 1, sinon k >= 2
    int startK = (y > 0 ? 1 : 2);
    for (int k = startK; ; k++) {
        int z = y + k * x;
        if (z > LIMIT) {
            break;
        }
        exploreGcds(z, x, steps + 1);
    }
}

int main() {
    for (int i = 1; i <= LIMIT; i++) {
        exploreGcds(i, 0, 0);
    }
    
    long long result = sum * 2 + (long long)LIMIT * (LIMIT + 1) / 2;
    cout << result << endl;
    return 0;
}
```
