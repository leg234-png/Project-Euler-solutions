# Nombres figurés cycliques

## Description du Problème
Les nombres triangulaires, carrés, pentagonaux, hexagonaux, heptagonaux et octogonaux sont des nombres figurés. Un ensemble ordonné de trois nombres à 4 chiffres : 8128, 2882, 8281, a trois propriétés intéressantes :
1. L'ensemble est cyclique : les deux derniers chiffres de chaque nombre sont les deux premiers chiffres du nombre suivant (y compris le dernier nombre avec le premier).
2. Chaque type polygonal : triangle (P₃,₁₂₇=8128), carré (P₄,₉₁=8281), et pentagonal (P₅,₄₄=2882), est représenté par un nombre différent dans l'ensemble.
3. C'est le seul ensemble de nombres à 4 chiffres avec cette propriété.

**Objectif** : Trouver la somme du seul ensemble ordonné de six nombres cycliques à 4 chiffres pour lequel chaque type polygonal : triangle, carré, pentagonal, hexagonal, heptagonal, et octogonal, est représenté par un nombre différent dans l'ensemble.

---

## Exemples
### Exemple 1
**Entrée** : Ensemble de 3 nombres
**Sortie** : `8128 + 2882 + 8281 = 19291`
**Explication** : L'ensemble {8128, 2882, 8281} est cyclique et représente les types triangle, carré et pentagonal.

---

## Approche de Résolution

### Stratégie
1. Générer tous les nombres figurés à 4 chiffres pour chaque type (triangle, carré, pentagonal, hexagonal, heptagonal, octogonal).
2. Organiser ces nombres par leurs deux premiers chiffres pour faciliter la recherche cyclique.
3. Utiliser une recherche récursive avec backtracking pour trouver un ensemble de 6 nombres cycliques, un de chaque type.
4. Vérifier que les nombres forment un cycle (les deux derniers chiffres d'un nombre correspondent aux deux premiers du suivant).

### Observations
- Les nombres figurés sont générés par des formules spécifiques :
  - Triangle : P₃,n = n(n+1)/2
  - Carré : P₄,n = n²
  - Pentagonal : P₅,n = n(3n-1)/2
  - Hexagonal : P₆,n = n(2n-1)
  - Heptagonal : P₇,n = n(5n-3)/2
  - Octogonal : P₈,n = n(3n-2)
- La recherche utilise un bitmask pour suivre les types déjà utilisés.
- On commence par un nombre triangulaire et on cherche à compléter le cycle.

### Complexité
- **Temporelle** : O(N × M), où N est le nombre de nombres figurés et M est le nombre de combinaisons possibles.
- **Spatiale** : O(N), pour stocker les nombres figurés organisés par préfixe.

---

## Solution en C++

```cpp
#include <iostream>
#include <vector>
#include <set>
#include <algorithm>

using namespace std;

vector<set<int>> numbers[9][100];

int figurateNumber(int sides, int n) {
    return n * ((sides - 2) * n - (sides - 4)) / 2;
}

int findSolutionSum(int begin, int current, int sidesUsed, int sum) {
    if (sidesUsed == 0x1F8) {  // Tous les types utilisés (3-8)
        if (current % 100 == begin / 100)
            return sum;
    } else {
        for (int sides = 4; sides <= 8; sides++) {
            if (((sidesUsed >> sides) & 1) != 0)
                continue;
            for (int num : numbers[sides][current % 100]) {
                int temp = findSolutionSum(begin, num, sidesUsed | (1 << sides), sum + num);
                if (temp != -1)
                    return temp;
            }
        }
    }
    return -1;
}

int main() {
    // Initialiser les tableaux
    for (int i = 0; i < 9; i++) {
        for (int j = 0; j < 100; j++) {
            numbers[i][j] = set<int>();
        }
    }
    
    // Générer les nombres figurés à 4 chiffres
    for (int sides = 3; sides <= 8; sides++) {
        for (int n = 1; ; n++) {
            int num = figurateNumber(sides, n);
            if (num >= 10000)
                break;
            if (num >= 1000)
                numbers[sides][num / 100].insert(num);
        }
    }
    
    // Rechercher la solution
    for (int i = 10; i < 100; i++) {
        for (int num : numbers[3][i]) {
            int temp = findSolutionSum(num, num, 1 << 3, num);
            if (temp != -1) {
                cout << temp << endl;
                return 0;
            }
        }
    }
    
    return 1;
}
```
