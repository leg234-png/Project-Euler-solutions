# Somme des Chiffres d'une Puissance

## Description du Problème
Un nombre de Googol (10^100) est un nombre massif : un suivi de cent zéros. 100^100 est presque inimaginablement grand : un suivi de deux cents zéros. Malgré leur taille, la somme des chiffres de chaque nombre n'est que 1.

En considérant les nombres naturels de la forme a^b, où a, b < 100, quelle est la somme maximale des chiffres ?

---

## Exemples
### Exemple 1
**Entrée** : a = 10, b = 100
**Sortie** : `1`
**Explication** : 10^100 = 1 suivi de 100 zéros, donc la somme des chiffres est 1.

---

## Approche de Résolution

### Stratégie
1. Pour chaque a de 1 à 99 :
   - Pour chaque b de 1 à 99 :
     - Calculer a^b en utilisant BigInteger.
     - Calculer la somme des chiffres.
     - Mettre à jour le maximum.
2. Retourner le maximum.

### Observations
- On doit utiliser BigInteger car a^b peut être très grand.
- On calcule la somme des chiffres en convertissant en chaîne de caractères.

### Complexité
- **Temporelle** : O(a × b × log(a^b)), où a = 99, b = 99.
- **Spatiale** : O(log(a^b)), pour stocker les BigInteger.

---

## Solution en C++

```cpp
#include <iostream>
#include <string>
#include <gmpxx.h>

int digitSum(const mpz_class& n) {
    std::string s = n.get_str();
    int sum = 0;
    for (char c : s) {
        sum += c - '0';
    }
    return sum;
}

int main() {
    int max = 0;
    for (int a = 1; a < 100; ++a) {
        for (int b = 1; b < 100; ++b) {
            mpz_class pow;
            mpz_pow_ui(pow.get_mpz_t(), mpz_class(a).get_mpz_t(), b);
            max = std::max(digitSum(pow), max);
        }
    }
    std::cout << max << std::endl;
    return 0;
}
```
