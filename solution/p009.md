# Triplet Pythagoricien Spécial

## Description du Problème
Un triplet pythagoricien est un ensemble de trois nombres naturels, a < b < c, pour lesquels :

a² + b² = c²

Par exemple, 3² + 4² = 9 + 16 = 25 = 5².

**Objectif** : Il existe exactement un triplet pythagoricien pour lequel a + b + c = 1000. Trouver le produit abc.

---

## Exemples
### Exemple 1
**Entrée** : Périmètre = 12
**Sortie** : `60`
**Explication** : Le triplet (3, 4, 5) vérifie 3² + 4² = 5² et 3 + 4 + 5 = 12. Le produit est 3×4×5 = 60.

---

## Approche de Résolution

### Stratégie
1. Parcourir toutes les valeurs possibles de a et b.
2. Pour chaque paire (a, b), calculer c = 1000 - a - b.
3. Vérifier si a² + b² = c² et si a < b < c.
4. Si la condition est satisfaite, retourner le produit a × b × c.

### Observations
- On peut optimiser en commençant par a = 1 et b = a + 1.
- On s'assure que c = 1000 - a - b est positif et que b < c.
- On peut arrêter la recherche dès qu'on trouve le triplet.
- Il existe des formules mathématiques pour générer des triplets pythagoriciens, mais pour ce problème, la recherche exhaustive est suffisante.

### Complexité
- **Temporelle** : O(n²), où `n` est le périmètre (1000).
- **Spatiale** : O(1), car on utilise seulement quelques variables.

---

## Solution en C++

```cpp
#include <iostream>

int main() {
    const int perimeter = 1000;
    
    for (int a = 1; a < perimeter; ++a) {
        for (int b = a + 1; b < perimeter; ++b) {
            int c = perimeter - a - b;
            if (b < c && a * a + b * b == c * c) {
                std::cout << (a * b * c) << std::endl;
                return 0;
            }
        }
    }
    
    return 0;
}
```
