# Paires de chiffres de cube

## Description du Problème
Chacune des six faces d'un cube a un chiffre différent (0 à 9) écrit dessus ; la même chose est faite pour un second cube. En plaçant les deux cubes côte à côte dans différentes positions, on peut former une variété de nombres à 2 chiffres.

En fait, en choisissant soigneusement les chiffres sur les deux cubes, il est possible d'afficher tous les nombres carrés inférieurs à cent : 01, 04, 09, 16, 25, 36, 49, 64, et 81.

Par exemple, une façon d'y parvenir est de placer {0,5,6,7,8,9} sur un cube et {1,2,3,4,8,9} sur l'autre cube.

Cependant, pour ce problème, on permettra au 6 ou au 9 d'être retournés à l'envers de sorte qu'un arrangement comme {0,5,6,7,8,9} et {1,2,3,4,6,7} permet d'afficher les neuf nombres carrés ; sinon il serait impossible d'obtenir 09.

En déterminant un arrangement distinct, on s'intéresse aux chiffres sur chaque cube, pas à l'ordre.
{1,2,3,4,5,6} est équivalent à {3,6,4,1,2,5}
{1,2,3,4,5,6} est distinct de {1,2,3,4,5,9}

Mais parce qu'on permet au 6 et au 9 d'être inversés, les deux ensembles distincts dans le dernier exemple représentent tous les deux l'ensemble étendu {1,2,3,4,5,6,9} dans le but de former des nombres à 2 chiffres.

**Objectif** : Combien d'arrangements distincts des deux cubes permettent d'afficher tous les nombres carrés ?

---

## Exemples
### Exemple 1
**Entrée** : Recherche d'arrangements
**Sortie** : `1217` (exemple)
**Explication** : On compte tous les arrangements distincts qui permettent d'afficher les 9 nombres carrés.

---

## Approche de Résolution

### Stratégie
1. Générer toutes les combinaisons possibles de 6 chiffres parmi 10 pour chaque cube.
2. Pour chaque paire de cubes, vérifier si tous les nombres carrés peuvent être formés.
3. Les nombres carrés sont : 01, 04, 09, 16, 25, 36, 49, 64, 81.
4. Traiter 6 et 9 comme équivalents (si un cube a 6, il peut aussi former 9, et vice versa).
5. Compter les arrangements valides.

### Observations
- Chaque cube a (10 choose 6) = 210 arrangements possibles.
- On utilise un bitmask pour représenter les chiffres sur chaque cube.
- Si un cube a 6 ou 9, on considère qu'il a les deux.
- On vérifie que chaque nombre carré peut être formé avec les deux cubes.

### Complexité
- **Temporelle** : O(2^10 × 2^10) = O(2^20), pour toutes les combinaisons possibles.
- **Spatiale** : O(1), utilisation d'espace constant.

---

## Solution en C++

```cpp
#include <iostream>
#include <vector>

using namespace std;

vector<vector<int>> SQUARES = {
    {0, 1}, {0, 4}, {0, 9}, {1, 6}, {2, 5},
    {3, 6}, {4, 9}, {6, 4}, {8, 1}
};

bool testBit(int x, int i) {
    return ((x >> i) & 1) != 0;
}

bool isArrangementValid(int a, int b) {
    // Traiter 6 et 9 comme équivalents
    if (testBit(a, 6) || testBit(a, 9))
        a |= (1 << 6) | (1 << 9);
    if (testBit(b, 6) || testBit(b, 9))
        b |= (1 << 6) | (1 << 9);
    
    for (const auto& sqr : SQUARES) {
        bool canForm = (testBit(a, sqr[0]) && testBit(b, sqr[1])) ||
                       (testBit(a, sqr[1]) && testBit(b, sqr[0]));
        if (!canForm)
            return false;
    }
    return true;
}

int bitCount(int x) {
    int count = 0;
    while (x != 0) {
        count += x & 1;
        x >>= 1;
    }
    return count;
}

int main() {
    int count = 0;
    for (int i = 0; i < (1 << 10); i++) {
        for (int j = i; j < (1 << 10); j++) {  // i <= j pour éviter les doublons
            if (bitCount(i) == 6 && bitCount(j) == 6 && isArrangementValid(i, j))
                count++;
        }
    }
    cout << count << endl;
    return 0;
}
```
