# Paires de diviseurs

## Description du Problème
Soit S(n) = ∑_{i=1}^n ∑_{j=1}^n i×j / gcd(i,j).

Soit E(m, n) le plus grand entier k tel que 2ᵏ divise S((pₘ)^n), où pₘ est le m-ième nombre premier.

Soit Q(n) = ∑_{i=1}^n E(904961, i).

Q(8) = 2714886.

**Objectif** : Évaluer Q(10¹²).

---

## Exemples
### Exemple 1
**Entrée** : `n = 8`
**Sortie** : `2714886`
**Explication** : Q(8) = 2714886.

---

## Approche de Résolution

### Stratégie
1. Observer que S((pₘ)^n) suit des patterns spécifiques.
2. S((pₘ)^1) suit 3^m - 2^m.
3. S((pₘ)^2) suit 6^m - 3^m.
4. S((pₘ)^n) = ((n+1)(n+2)/2)^m - (n+1)^m.
5. E(m, n) = plus grande puissance de 2 divisant S((pₘ)^n).
6. Pour k > 0, n doit être impair (donc n+1 est pair).
7. Si n+1 est divisible par 4, alors on peut calculer la puissance de 2.
8. Utiliser la formule pour la somme partielle.

### Observations
- Il y a des patterns clairs dans les valeurs de S((pₘ)^n).
- On peut calculer E(m, n) directement sans calculer S((pₘ)^n) explicitement.
- La formule de somme partielle permet de calculer Q(n) efficacement.

### Complexité
- **Temporelle** : O(n) pour calculer Q(n).
- **Spatiale** : O(1).

---

## Solution en C++

```cpp
#include <iostream>
#include <cmath>

using namespace std;

int highest_power_of_2(long long n) {
    int count = 0;
    while (n % 2 == 0) {
        n /= 2;
        count++;
    }
    return count;
}

long long compute(long long n) {
    long long total = 0;
    int m = 904961;
    
    for (long long i = 1; i <= n; i++) {
        if (i % 2 == 1) {
            // n est impair, donc n+1 est pair
            if ((i + 1) % 4 == 0) {
                long long base = (i + 1) / 2;
                int power = highest_power_of_2(base);
                total += (long long)power * m;
            }
        }
    }
    
    return total;
}

int main() {
    cout << compute(1000000000000LL) << endl;
    return 0;
}
```
