# Arbre récursif

## Description du Problème
Soit f(n, k) la somme des numéros de nœuds le long du chemin reliant la racine de T_n au nœud k, incluant la racine et le nœud k.

Par exemple, f(6, 1) = 6+5+1 = 12 et f(10, 3) = 29.

**Objectif** : Trouver f(10¹⁷, 9¹⁷).

---

## Exemples
### Exemple 1
**Entrée** : `n = 6, k = 1`
**Sortie** : `12`
**Explication** : f(6, 1) = 6+5+1 = 12.

---

## Approche de Résolution

### Stratégie
1. Après avoir dessiné quelques arbres, quelques patterns émergent. Par exemple, la hauteur de l'arbre est log₂(n) + 1.
2. Les enfants du nœud n sont toujours [n-1, n-2, n-4, n-8, n-16, etc].
3. Les enfants de n-1 sont toujours [n-3, n-5, n-9, etc].
4. Essentiellement, il y a un pattern lié à 2^n.
5. Si y est le parent de x, alors y - x = 2^k, où k est le plus grand tel que x + 2^k ≤ n.
6. Si x = n, il n'a pas de parents.
7. Utiliser cette propriété pour calculer f(n, k).

### Observations
- Il y a un pattern clair dans la structure de l'arbre.
- On peut calculer le chemin de la racine à n'importe quel nœud efficacement.
- La formule permet de calculer sans construire l'arbre entier.

### Complexité
- **Temporelle** : O(log n) pour calculer f(n, k).
- **Spatiale** : O(1).

---

## Solution en C++

```cpp
#include <iostream>
#include <cmath>

using namespace std;

long long f(long long n, long long x) {
    long long total = x;
    long long curr = x;
    
    while (curr != n) {
        long long k = (long long)log2(n - curr);
        curr += (1LL << k);
        total += curr;
    }
    
    return total;
}

int main() {
    long long n = 1;
    for (int i = 0; i < 17; i++) {
        n *= 10;
    }
    
    long long k = 1;
    for (int i = 0; i < 17; i++) {
        k *= 9;
    }
    
    cout << f(n, k) << endl;
    return 0;
}
```
