# Special Subset Sums: Testing

## Description du Problème
Soit S(A) la somme des éléments de l'ensemble A de taille n. Nous l'appellerons un ensemble de somme spéciale si pour deux sous-ensembles non vides disjoints quelconques, B et C, les propriétés suivantes sont vraies :
1. S(B) ≠ S(C) ; c'est-à-dire que les sommes des sous-ensembles ne peuvent pas être égales.
2. Si B contient plus d'éléments que C, alors S(B) > S(C).

Par exemple, {81, 88, 75, 42, 87, 84, 86, 65} n'est pas un ensemble de somme spéciale car 65 + 87 + 88 = 75 + 81 + 84, alors que {157, 150, 164, 119, 79, 159, 161, 139, 158} satisfait les deux règles pour toutes les combinaisons possibles de paires de sous-ensembles et S(A) = 1286.

En utilisant sets.txt, un fichier texte de 4K avec cent ensembles contenant sept à douze éléments (les deux exemples donnés ci-dessus sont les deux premiers ensembles du fichier), identifier tous les ensembles de somme spéciale, A₁, A₂, …, Aₖ, et trouver la valeur de S(A₁) + S(A₂) + ⋯ + S(Aₖ).

**Objectif** : Trouver la somme des sommes de tous les ensembles de somme spéciale dans le fichier.

---

## Exemples
### Exemple 1
**Entrée** : `{81, 88, 75, 42, 87, 84, 86, 65}`
**Sortie** : `false` (n'est pas un ensemble de somme spéciale)
**Explication** : 65 + 87 + 88 = 75 + 81 + 84, violant la propriété (i).

### Exemple 2
**Entrée** : `{157, 150, 164, 119, 79, 159, 161, 139, 158}`
**Sortie** : `true` (est un ensemble de somme spéciale)
**Explication** : S(A) = 1286 et toutes les propriétés sont satisfaites.

---

## Approche de Résolution

### Stratégie
1. Lire tous les ensembles depuis le fichier.
2. Pour chaque ensemble, vérifier les deux propriétés :
   - Générer tous les sous-ensembles et vérifier que leurs sommes sont distinctes.
   - Vérifier que si |B| > |C|, alors S(B) > S(C).
3. Si un ensemble satisfait les deux propriétés, ajouter sa somme au total.

### Observations
- On peut utiliser un masque de bits pour générer tous les sous-ensembles.
- On maintient des tableaux pour les sommes min/max par taille de sous-ensemble.
- On utilise un HashSet pour vérifier l'unicité des sommes.

### Complexité
- **Temporelle** : O(n × 2ⁿ) pour chaque ensemble, où n est la taille de l'ensemble.
- **Spatiale** : O(2ⁿ) pour stocker les informations sur les sous-ensembles.

## Solution en C++

```cpp
#include <iostream>
#include <vector>
#include <set>
#include <algorithm>
#include <climits>
#include <fstream>
#include <sstream>

bool isSpecialSumSet(const std::vector<int>& set) {
    std::set<int> sumsSeen;
    std::vector<int> minSum(set.size() + 1, INT_MAX);
    std::vector<int> maxSum(set.size() + 1, 0);
    
    // Générer tous les sous-ensembles
    for (int i = 1; i < (1 << set.size()); i++) {
        int size = __builtin_popcount(i);
        int sum = 0;
        for (int j = 0; j < set.size(); j++) {
            if (i & (1 << j)) {
                sum += set[j];
            }
        }
        
        // Vérifier la propriété (i) : sommes distinctes
        if (!sumsSeen.insert(sum).second) {
            return false;
        }
        
        minSum[size] = std::min(minSum[size], sum);
        maxSum[size] = std::max(maxSum[size], sum);
    }
    
    // Vérifier la propriété (ii) : si |B| > |C|, alors S(B) > S(C)
    for (int i = 0; i < set.size(); i++) {
        if (maxSum[i] >= minSum[i + 1]) {
            return false;
        }
    }
    
    return true;
}
```
