# Investigating the behaviour of a recursively defined sequence

## Description du Problème
On définit la fonction `f(x) = floor(2^(30.403243784 - x²)) × 10^-9`.

La suite `u_n` est définie par :
- `u_0 = -1`
- `u_(n+1) = f(u_n)`

**Objectif** : Trouver `u_n + u_(n+1)` pour `n = 10^12`. Donner la réponse avec 9 chiffres après la virgule.

---

## Exemples
### Exemple 1
**Entrée** : `n = 0`
**Sortie** : `u_0 = -1, u_1 = f(-1)`
**Explication** : On calcule les premiers termes de la suite.

---

## Approche de Résolution

### Stratégie
1. Utiliser l'algorithme de détection de cycle de Floyd (tortue et lièvre).
2. Détecter quand la suite entre dans un cycle.
3. Une fois le cycle détecté :
   - Calculer la longueur du cycle.
   - Avancer de `(ITERATIONS - index_début_cycle) % longueur_cycle` itérations.
   - Calculer `u_n + u_(n+1)`.

### Observations
- La suite finit par entrer dans un cycle.
- On peut détecter le cycle en comparant `u_i` et `u_(2i)`.
- Une fois le cycle détecté, on peut sauter directement à l'itération désirée.

### Complexité
- **Temporelle : O(λ + μ)**, où λ est la longueur du cycle et μ est la longueur de la pré-période.
- **Spatiale : O(1)**.

---

## Solution en C++

```cpp
#include <iostream>
#include <iomanip>
#include <cmath>

using namespace std;

const long long ITERATIONS = 1000000000000LL;

double f(double x) {
    return floor(pow(2.0, 30.403243784 - x * x)) / 1e9;
}

int main() {
    double x = -1.0;
    double y = -1.0;
    long long i = 0;
    
    // Floyd's cycle-finding algorithm
    for (; i < ITERATIONS; i++) {
        if (i > 0 && x == y)
            break;
        
        x = f(x);
        y = f(f(y));
    }
    
    // Advance by remaining iterations
    long long remain = (ITERATIONS - i) % i;
    for (; remain > 0; remain--)
        x = f(x);
    
    double answer = x + f(x);
    answer = floor(answer * 1e9) / 1e9;  // Truncate to 9 digits
    
    cout << fixed << setprecision(9) << answer << endl;
    return 0;
}
```
