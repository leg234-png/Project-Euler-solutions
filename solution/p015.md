# Chemins dans une Grille

## Description du Problème
En partant du coin supérieur gauche d'une grille 2×2 et en ne pouvant se déplacer que vers la droite et vers le bas, il y a exactement 6 chemins pour atteindre le coin inférieur droit.

**Objectif** : Combien y a-t-il de chemins dans une grille 20×20 ?

---

## Exemples
### Exemple 1
**Entrée** : Grille 2×2
**Sortie** : `6`
**Explication** : Il y a 6 chemins possibles pour aller du coin supérieur gauche au coin inférieur droit dans une grille 2×2.

---

## Approche de Résolution

### Stratégie
C'est un problème classique de combinatoire. Pour aller du coin supérieur gauche au coin inférieur droit d'une grille n×n, il faut faire exactement n déplacements vers la droite et n déplacements vers le bas, dans un certain ordre.

Le nombre de chemins est donné par le coefficient binomial : C(2n, n) = (2n)! / (n! × n!)

### Observations
- Pour une grille n×n, on doit faire n déplacements vers la droite et n vers le bas.
- L'ordre des déplacements n'a pas d'importance (ils sont indistinguables).
- Le nombre de façons d'arranger ces déplacements est C(2n, n).
- Pour n = 20, on calcule C(40, 20) = 40! / (20! × 20!).
- On peut calculer le coefficient binomial de manière itérative pour éviter les débordements.

### Complexité
- **Temporelle** : O(n), où `n` = 20.
- **Spatiale** : O(1), car on utilise seulement quelques variables.

---

## Solution en C++

```cpp
#include <iostream>

long long binomial(int n, int k) {
    if (k > n - k) {
        k = n - k;  // Optimisation : C(n, k) = C(n, n-k)
    }
    
    long long result = 1;
    for (int i = 0; i < k; ++i) {
        result = result * (n - i) / (i + 1);
    }
    
    return result;
}

int main() {
    std::cout << binomial(40, 20) << std::endl;
    return 0;
}
```
