# Progressive Square Numbers

## Description du Problème
Un entier positif, n, est divisé par d et le quotient et le reste sont q et r respectivement. De plus, d, q et r sont des termes entiers positifs consécutifs dans une séquence géométrique, mais pas nécessairement dans cet ordre.

Par exemple, 58 divisé par 6 a un quotient 9 et un reste 4. On peut aussi voir que 4, 6, 9 sont des termes consécutifs dans une séquence géométrique (rapport commun 3/2).

Nous appellerons de tels nombres, n, progressifs.

Certains nombres progressifs, tels que 9 et 10404 = 102², se trouvent également être des carrés parfaits. La somme de tous les carrés parfaits progressifs en dessous de cent mille est 124657.

**Objectif** : Trouver la somme de tous les carrés parfaits progressifs en dessous d'un billion (10¹²).

---

## Exemples
### Exemple 1
**Entrée** : `n = 58`
**Sortie** : `false` (n'est pas un carré)
**Explication** : 58 est progressif mais n'est pas un carré parfait.

### Exemple 2
**Entrée** : `n = 10404`
**Sortie** : `true`
**Explication** : 10404 = 102² et est progressif.

---

## Approche de Résolution

### Stratégie
1. Pour chaque carré parfait n = k², vérifier s'il est progressif.
2. Pour chaque diviseur d de n, calculer q = n/d et r = n mod d.
3. Vérifier si d, q, r forment une séquence géométrique (dans n'importe quel ordre).
4. Si oui, ajouter n à la somme.

### Observations
- On peut limiter la recherche en utilisant des contraintes sur les séquences géométriques.
- On doit vérifier tous les ordres possibles de d, q, r.

### Complexité
- **Temporelle** : O(n^(3/2)) pour tester tous les carrés et leurs diviseurs.
- **Spatiale** : O(1).

bool isProgressive(long long n) {
    for (long long d = 1; d * d <= n; d++) {
        if (n % d == 0) {

int main() {
    const long long LIMIT = 1000000000000LL;
    long long sum = 0;
    
    for (long long k = 1; k * k < LIMIT; k++) {
        long long n = k * k;
        if (isProgressive(n)) {
            sum += n;
        }
    }
    
    std::cout << sum << std::endl;

}
