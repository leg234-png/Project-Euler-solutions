# Palindromes divisibles

## Description du Problème
Les nombres 545, 5995 et 15151 sont les trois plus petits palindromes divisibles par 109. Il y a neuf palindromes inférieurs à 100000 qui sont divisibles par 109.

**Objectif** : Combien de palindromes inférieurs à 10³² sont divisibles par 10000019 ?

---

## Exemples
### Exemple 1
**Entrée** : `diviseur = 109, limite = 100000`
**Sortie** : `9`
**Explication** : Il y a 9 palindromes < 100000 divisibles par 109.

---

## Approche de Résolution

### Stratégie
1. Un palindrome de longueur n peut être écrit comme a×10^(n-1) + b×10^(n-2) + ... + b×10 + a.
2. Pour un palindrome pair : x = a×10^4 + b×10^3 + c×10^2 + b×10 + a.
3. Pour un palindrome impair : x = a×10^4 + b×10^3 + c×10^2 + d×10 + c×10^0 + b×10^1 + a×10^2.
4. Calculer x mod d pour chaque configuration de chiffres.
5. Compter combien sont ≡ 0 (mod d).
6. Utiliser la programmation dynamique pour compter efficacement.

### Observations
- On peut générer tous les palindromes de manière systématique.
- On peut utiliser l'arithmétique modulaire pour vérifier la divisibilité.
- Il y a des patterns dans les palindromes qui peuvent être exploités.

### Complexité
- **Temporelle** : O(10^L) où L est la longueur maximale du palindrome.
- **Spatiale** : O(1).

---

long long compute(long long limit, long long divisor) {
    long long total = 0;
    int max_length = to_string(limit).length();
