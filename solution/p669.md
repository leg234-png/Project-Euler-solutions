# Le banquet du roi

## Description du Problème
Les Chevaliers de l'Ordre de Fibonacci préparent un grand festin pour leur roi. Il y a n chevaliers, et chaque chevalier se voit attribuer un nombre distinct de 1 à n.

Les chevaliers doivent s'asseoir autour d'une table circulaire de manière à ce que la somme des nombres de deux chevaliers adjacents soit toujours un nombre de Fibonacci.

Par exemple, pour n=7, un arrangement possible est [4,1,7,6,2,3,5]. Remarquez que les sommes 4+1, 1+7, 7+6, 6+2, 2+3, et 3+5 sont toutes des nombres de Fibonacci.

Le roi préfère toujours un arrangement où le chevalier à sa gauche a un nombre plus petit que le chevalier à sa droite. Avec cette règle supplémentaire, l'arrangement ci-dessus est unique pour n=7, et le chevalier assis sur la 3ème chaise depuis la gauche du roi est le chevalier numéro 7.

Plus tard, plusieurs nouveaux chevaliers sont nommés à l'Ordre, donnant 34 chevaliers et chaises en plus du trône du roi. Les chevaliers déterminent finalement qu'il y a un arrangement unique pour n=34 satisfaisant les règles ci-dessus, et cette fois le chevalier numéro 30 est assis sur la 3ème chaise depuis la gauche du roi.

**Objectif** : Soit F_{n+1} = 160500643816367088, F_n = 99194853094755497, F_{n-1} = 61305790721611591. Trouver le chevalier assis sur la 10¹⁶-ième chaise depuis la gauche du roi.

---

## Exemples
### Exemple 1
**Entrée** : `n = 7`
**Sortie** : `7`
**Explication** : Le chevalier sur la 3ème chaise est le numéro 7.

---

## Approche de Résolution

### Stratégie
1. Observer les patterns entre l'ajout du terme x et x+1.
2. Soit x = F_{n+1}, y = F_n, z = F_{n-1}.
3. Ils alternent entre x, z et y.
4. Si on est actuellement à y, alors la prochaine somme serait x ou z.
5. Si on est actuellement à x ou z, alors la prochaine somme serait y.
6. Utiliser les propriétés modulaires pour calculer la position finale.

### Observations
- Il y a des patterns clairs liés aux nombres de Fibonacci.
- On peut utiliser l'arithmétique modulaire pour calculer efficacement.
- La position peut être calculée sans générer tout l'arrangement.

### Complexité
- **Temporelle** : O(log n) pour le calcul modulaire.
- **Spatiale** : O(1).

## Solution en C++

```cpp
#include <iostream>
#include <cmath>

using namespace std;

long long compute(long long position) {
    long long F_n_plus_1 = 160500643816367088LL;
    long long F_n = 99194853094755497LL;
    long long F_n_minus_1 = 61305790721611591LL;
    
    // Calculer la position en utilisant les patterns observés
    long long moves = position - F_n;
    long long apply_x_to_minus_x = moves / 2;
    long long apply_x_to_z_minus_a = (moves - 1) / 2;
    
    long long result = (apply_x_to_minus_x * F_n_minus_1) % F_n;
    return result;
}

int main() {
    long long position = 10000000000000000LL;
    cout << compute(position) << endl;
    return 0;
}
```
