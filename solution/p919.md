# Triangles chanceux

## Description du Problème
Un triangle avec des côtés entiers a, b, c (avec a ≤ b ≤ c) est appelé chanceux si a² + b² + c² = 3ab.

Définissons S(P) comme la somme de a+b+c sur tous les triangles chanceux avec des côtés a ≤ b ≤ c et un périmètre ne dépassant pas P.

Par exemple, S(10) = 24, provenant de trois triangles avec des côtés (1,2,2), (2,3,4), et (2,4,4). On vous donne aussi S(100) = 3331.

**Objectif** : Trouver S(10⁶).

---

## Exemples
### Exemple 1
**Entrée** : `P = 10`
**Sortie** : `24`
**Explication** : S(10) = 24.

---

## Approche de Résolution

### Stratégie
1. Résoudre a² + b² + c² = 3ab pour trouver les relations entre a, b, c.
2. Réarranger : c² = 3ab - a² - b² = (a+b)² - 4ab + ab = (a+b)² - 3ab.
3. Pour que c soit un entier, 3ab - a² - b² doit être un carré parfait.
4. Itérer sur a et b, vérifier si c = √(3ab - a² - b²) est un entier.
5. Vérifier que a ≤ b ≤ c et a + b + c ≤ P.
6. Sommer tous les périmètres valides.

### Observations
- Il y a des contraintes géométriques sur les triangles.
- On peut optimiser en limitant la plage de recherche pour a et b.
- Il y a probablement des patterns dans les solutions.

### Complexité
- **Temporelle** : O(P²) dans le pire cas.
- **Spatiale** : O(1).

---

## Solution en C++

```cpp
#include <iostream>
#include <cmath>

using namespace std;

long long compute(long long P) {
    long long total = 0;
    
    for (long long a = 1; a <= P / 3; a++) {
        for (long long b = a; b <= P / 2; b++) {
            long long discriminant = 3 * a * b - a * a - b * b;
            if (discriminant < 0) continue;
            
            long long c_sq = discriminant;
            long long c = sqrt(c_sq);
            
            if (c * c == c_sq && c >= b && a + b + c <= P) {
                total += a + b + c;
            }
        }
    }
    
    return total;
}

int main() {
    long long P = 1;
    for (int i = 0; i < 6; i++) {
        P *= 10;
    }
    cout << compute(P) << endl;
    return 0;
}
```
