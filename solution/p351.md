# Vergers hexagonaux

## Description du Problème
Soit H(n) le nombre de points cachés du centre dans un verger hexagonal d'ordre n.

H(5)=30. H(10)=138. H(1000)=1177848.

**Objectif** : Trouver H(10⁸).

---

## Exemples
### Exemple 1
**Entrée** : `n = 5`
**Sortie** : `30`
**Explication** : H(5) = 30.

---

## Approche de Résolution

### Stratégie
1. On n'a besoin de regarder qu'une section (triangle) de l'hexagone, puis multiplier par 6.
2. Du centre de l'hexagone, on trace une ligne vers chaque point.
3. Les angles suivent un pattern : couche 1 a angles 0/1, 1/1 ; couche 2 a angles 0/2, 1/2, 2/2 ; etc.
4. Le problème se réduit à trouver le nombre de fractions qui ne sont pas sous forme réduite.
5. Sur la couche n, il y a n fractions, et le nombre de fractions non réduites est n - φ(n).
6. Réponse = 6 × ∑ᵢ₌₁ⁿ (i - φ(i)) = 3n(n+1) - 6∑ᵢ₌₁ⁿ φ(i).

### Observations
- On utilise la fonction sommatoire de la fonction d'Euler.
- Il existe des méthodes efficaces pour calculer ∑φ(i) jusqu'à n.
- On peut utiliser un crible modifié pour calculer la somme.

### Complexité
- **Temporelle** : O(n log log n) pour le crible et le calcul de la somme.
- **Spatiale** : O(n) pour le crible.

---

## Solution en C++

```cpp
#include <iostream>
#include <vector>
#include <cmath>

using namespace std;

long long totient_summatory(long long n) {
    vector<long long> phi(n + 1);
    for (long long i = 0; i <= n; i++) {
        phi[i] = i;
    }
    
    for (long long i = 2; i <= n; i++) {
        if (phi[i] == i) {
            for (long long j = i; j <= n; j += i) {
                phi[j] -= phi[j] / i;
            }
        }
    }
    
    long long sum = 0;
    for (long long i = 1; i <= n; i++) {
        sum += phi[i];
    }
    return sum;
}

long long compute(long long n) {
    long long triangular_sum = 3 * n * (n + 1);
    long long totient_sum = 6 * totient_summatory(n);
    return triangular_sum - totient_sum;
}

int main() {
    cout << compute(100000000) << endl;
    return 0;
}
```
