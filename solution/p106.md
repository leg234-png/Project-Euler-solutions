# Special Subset Sums: Meta-testing

## Description du Problème
Soit S(A) la somme des éléments de l'ensemble A de taille n. Nous l'appellerons un ensemble de somme spéciale si pour deux sous-ensembles non vides disjoints quelconques, B et C, les propriétés suivantes sont vraies :
1. S(B) ≠ S(C) ; c'est-à-dire que les sommes des sous-ensembles ne peuvent pas être égales.
2. Si B contient plus d'éléments que C, alors S(B) > S(C).

Pour ce problème, nous supposerons qu'un ensemble donné contient n éléments strictement croissants et qu'il satisfait déjà la deuxième règle.

Étonnamment, parmi les 25 paires de sous-ensembles possibles qui peuvent être obtenues d'un ensemble pour lequel n = 4, seulement 1 de ces paires doit être testée pour l'égalité (première règle). De même, lorsque n = 7, seulement 70 des 966 paires de sous-ensembles doivent être testées.

**Objectif** : Pour n = 12, combien des 261625 paires de sous-ensembles qui peuvent être obtenues doivent être testées pour l'égalité ?

---

## Exemples
### Exemple 1
**Entrée** : `n = 4`
**Sortie** : `1`
**Explication** : Seulement 1 paire doit être testée parmi les 25 possibles.

### Exemple 2
**Entrée** : `n = 7`
**Sortie** : `70`
**Explication** : Seulement 70 paires doivent être testées parmi les 966 possibles.

---

## Approche de Résolution

### Stratégie
1. On ne doit tester que les paires de sous-ensembles de même taille (car la propriété (ii) garantit l'inégalité si les tailles diffèrent).
2. Pour chaque taille k ≥ 2, compter combien de paires de sous-ensembles de taille k doivent être testées.
3. Utiliser la formule combinatoire : pour chaque k de 2 à n/2, le nombre de paires à tester est :
   - (n choisir 2k) × [(2k choisir k) / 2 - (2k choisir k) / (k + 1)]
4. Sommer ces valeurs pour obtenir le total.

### Observations
- Les paires qui peuvent être "appariées" (chaque élément de B est inférieur à un élément correspondant de C) n'ont pas besoin d'être testées.
- Le nombre de paires appariables correspond aux nombres de Catalan.
- La formule finale utilise les coefficients binomiaux.

### Complexité
- **Temporelle** : O(n²) pour calculer les coefficients binomiaux et la somme.
- **Spatiale** : O(1).

---

## Solution en C++

```cpp
#include <iostream>
#include <gmpxx.h>

mpz_class binomial(int n, int k) {
    if (k > n - k) k = n - k;
    mpz_class result = 1;
    for (int i = 0; i < k; i++) {
        result = result * (n - i) / (i + 1);
    }
    return result;
}

int main() {
    const int SET_SIZE = 12;
    mpz_class ans = 0;
    
    for (int i = 2; i * 2 <= SET_SIZE; i++) {
        mpz_class x = binomial(SET_SIZE, i * 2);
        mpz_class y = binomial(i * 2, i) / 2;
        mpz_class z = binomial(i * 2, i) / (i + 1);  // Nombre de Catalan
        ans += x * (y - z);
    }
    
    std::cout << ans << std::endl;
    return 0;
}
```
