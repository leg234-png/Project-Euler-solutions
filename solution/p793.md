# Médiane de produits

## Description du Problème
Soit S₀ = 290797.

S_{n+1} = S_n² mod 50515093.

Soit M(n) la médiane des produits deux à deux S_i S_j pour 0 ≤ i < j < n.

On vous donne M(3) = 3878983057768 et M(10³) = 492700616748525.

**Objectif** : Trouver M(10⁶).

---

## Exemples
### Exemple 1
**Entrée** : `n = 3`
**Sortie** : `3878983057768`
**Explication** : M(3) = 3878983057768.

---

## Approche de Résolution

### Stratégie
1. Au lieu de calculer directement la médiane en listant tous les éléments (prendrait n² + n log(n) temps), faire quelque chose de plus intelligent.
2. Trier d'abord l'ensemble S = {S₀, ..., S_{n-1}}.
3. Utiliser une recherche binaire : soit low = S₀×S₁, high = S_{n-2}×S_{n-1}, et m le milieu, soit m notre estimation initiale de la médiane.
4. Il y aura nC2 = n(n-1)/2 produits deux à deux, donc s'il y a n(n-1)/4 produits qui sont inférieurs à m, alors m est la médiane.
5. Parcourir l'ensemble S une seule fois, ce qui est crucial pour réduire la complexité temporelle.
6. Disons qu'on est sur l'élément S_i, alors on trouve l'élément maximal j tel que S_i×S_j ≤ m, on peut faire cela avec une recherche binaire dans S en utilisant m/S_i.
7. Une fois qu'on a trouvé j, cela signifie qu'il y aura j - i produits qui ne dépassent pas m.
8. Une fois qu'on a trouvé le total de produits inférieurs à m, si le nombre est supérieur à notre objectif (n(n-1)/4), alors on sait que cette estimation est trop élevée, on pose high = mid - 1, sinon, si on a trop peu, alors posons lo = mid.
9. Après la recherche binaire, on aura trouvé notre médiane.

### Observations
- On peut utiliser une recherche binaire pour trouver la médiane sans générer tous les produits.
- La complexité est réduite à O(n² log(max_product)).
- On peut optimiser en utilisant des techniques de tri et de recherche.

### Complexité
- **Temporelle** : O(n² log(max_product)) avec recherche binaire.
- **Spatiale** : O(n) pour stocker S.

## Solution en C++

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <cmath>

using namespace std;

vector<long long> generate_S(int n) {
    vector<long long> S;
    long long s0 = 290797;
    S.push_back(s0);
    
    for (int i = 1; i < n; i++) {
        s0 = (s0 * s0) % 50515093;
        S.push_back(s0);
    }
    
    return S;
}

long long M(int n) {
    vector<long long> S = generate_S(n);
    sort(S.begin(), S.end());
    
    long long low = S[0] * S[1];
    long long high = S[n-2] * S[n-1];
    long long target = (long long)n * (n - 1) / 4;
    
    while (low < high) {
        long long mid = (low + high) / 2;
        long long count = 0;
        
        for (int i = 0; i < n; i++) {
            long long max_j = mid / S[i];
            int j = upper_bound(S.begin() + i + 1, S.end(), max_j) - S.begin();
            count += j - i - 1;
        }
        
        if (count > target) {
            high = mid;
        } else {
            low = mid + 1;
        }
    }
    
    return low;
}

int main() {
    cout << M(1000000) << endl;
    return 0;
}
```
