# Special Subset Sums: Optimum

## Description du Problème
Soit S(A) la somme des éléments de l'ensemble A de taille n. Nous l'appellerons un ensemble de somme spéciale si pour deux sous-ensembles non vides disjoints quelconques, B et C, les propriétés suivantes sont vraies :
1. S(B) ≠ S(C) ; c'est-à-dire que les sommes des sous-ensembles ne peuvent pas être égales.
2. Si B contient plus d'éléments que C, alors S(B) > S(C).

Si S(A) est minimisé pour un n donné, nous l'appellerons un ensemble de somme spéciale optimal. Les cinq premiers ensembles de somme spéciale optimaux sont donnés ci-dessous.

- n = 1 : {1}
- n = 2 : {1, 2}
- n = 3 : {2, 3, 4}
- n = 4 : {3, 5, 6, 7}
- n = 5 : {6, 9, 11, 12, 13}

Il semble que pour un ensemble optimal donné, A = {a₁, a₂, …, aₙ}, l'ensemble optimal suivant soit de la forme B = {b, a₁ + b, a₂ + b, …, aₙ + b}, où b est l'élément "milieu" de la ligne précédente.

En appliquant cette "règle", nous nous attendrions à ce que l'ensemble optimal pour n=6 soit A={11,17,20,22,23,24}, avec S(A)=117. Cependant, ce n'est pas l'ensemble optimal, car nous avons simplement appliqué un algorithme pour fournir un ensemble quasi optimal. L'ensemble optimal pour n=6 est A={11,18,19,20,22,25}, avec S(A)=115 et la chaîne d'ensemble correspondante : 111819202225.

**Objectif** : Étant donné que A est un ensemble de somme spéciale optimal pour n=7, trouver sa chaîne d'ensemble.

## Approche de Résolution

### Stratégie
1. Utiliser une recherche avec backtracking pour trouver l'ensemble optimal.
2. Pour chaque candidat, vérifier les deux propriétés :
   - Toutes les sommes de sous-ensembles sont distinctes.
   - Si |B| > |C|, alors S(B) > S(C).
3. Minimiser la somme totale tout en respectant les contraintes.

### Observations
- On peut utiliser une recherche en profondeur avec des optimisations.
- On maintient des tableaux pour suivre les sommes possibles et les sommes min/max par taille de sous-ensemble.
- On peut élaguer la recherche en vérifiant les propriétés au fur et à mesure.

### Complexité
- **Temporelle** : Exponentielle dans le pire cas, mais avec des optimisations, c'est gérable pour n=7.
- **Spatiale** : O(2ⁿ) pour stocker les informations sur les sous-ensembles.
