# Paper sheets of standard sizes: an expected-value problem

## Description du Problème
Un imprimeur utilise une machine qui prend une feuille de papier de taille A5 et la coupe en deux pour produire deux feuilles de taille A6. Chaque feuille ainsi produite a une taille de 1 à 5 (A1 à A5).

Lorsqu'une feuille de taille k (1 ≤ k ≤ 5) est utilisée, elle est coupée en deux pour produire deux feuilles de taille k+1, jusqu'à k+1 = 5.

Le processus commence avec une seule feuille de taille 1 (A5).

**Objectif** : Trouver le nombre attendu de fois qu'une seule feuille est présente dans l'enveloppe (c'est-à-dire le nombre de fois où il n'y a qu'une seule feuille dans l'enveloppe), en excluant le début et la fin du processus.

---

## Exemples
### Exemple 1
**Entrée** : Processus avec une feuille de taille 1
**Sortie** : Nombre attendu de fois qu'une seule feuille est présente
**Explication** : On simule le processus et on compte le nombre de fois où il y a exactement une feuille dans l'enveloppe.

---

## Approche de Résolution

### Stratégie
1. Utiliser la programmation dynamique avec mémoïsation pour calculer le nombre attendu de "singles" (moments où il y a exactement une feuille).
2. `getExpectedSingles(state)` : calcule le nombre attendu de singles à partir d'un état donné (liste des tailles de feuilles).
3. Pour chaque état :
   - Si l'état est vide, retourner 0.
   - Pour chaque feuille de taille k dans l'état :
     - Retirer cette feuille.
     - Ajouter deux feuilles de taille k+1 (si k+1 ≤ 5).
     - Trier l'état résultant.
     - Récursivement calculer le nombre attendu de singles pour le nouvel état.
   - Moyenner les résultats pour toutes les feuilles possibles.
   - Si l'état contient exactement une feuille, ajouter 1 au résultat (c'est un "single").
4. Soustraire 2 du résultat final (exclure le début et la fin).

### Observations
- L'état est représenté comme une liste triée de tailles de feuilles.
- La mémoïsation est cruciale pour éviter les recalculs.
- Chaque transition est équiprobable parmi toutes les feuilles disponibles.

### Complexité
- **Temporelle** : O(S × log S), où S est le nombre d'états possibles.
- **Spatiale** : O(S) pour la mémoïsation.

---

## Solution en C++

```cpp
#include <iostream>
#include <vector>
#include <map>
#include <algorithm>
#include <iomanip>

using namespace std;

map<vector<int>, double> expectedSingles;

double getExpectedSingles(vector<int> state) {
    sort(state.begin(), state.end());
    
    if (expectedSingles.count(state))
        return expectedSingles[state];
    
    double result = 0.0;
    if (!state.empty()) {
        for (size_t i = 0; i < state.size(); i++) {
            vector<int> newState = state;
            int sheet = newState[i];
            newState.erase(newState.begin() + i);
            
            for (int j = sheet + 1; j <= 5; j++)
                newState.push_back(j);
            
            result += getExpectedSingles(newState);
        }
        result /= state.size();
        if (state.size() == 1)
            result += 1.0;
    }
    
    expectedSingles[state] = result;
    return result;
}

int main() {
    vector<int> startState = {1};
    double result = getExpectedSingles(startState) - 2.0;
    cout << fixed << setprecision(6) << result << endl;
    return 0;
}
```
