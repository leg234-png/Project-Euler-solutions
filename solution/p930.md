# Le rassemblement

## Description du Problème
On a n bols disposés en cercle, numérotés de 0 à n-1. Initialement, m balles sont distribuées parmi les bols.

Le processus suivant est répété :
1. Choisir une balle au hasard parmi toutes les balles.
2. Choisir une direction (gauche ou droite) au hasard.
3. Déplacer la balle choisie vers le bol voisin dans la direction choisie.
4. Retourner à l'étape 1.

Ce processus s'arrête lorsque toutes les m balles sont situées dans le même bol. Notez que cela peut être après zéro étapes, si les balles ont été initialement distribuées toutes dans le même bol.

Soit F(n, m) le nombre attendu de fois où on déplace une balle avant que le processus ne s'arrête. Par exemple, F(2, 2) = 1/2, F(3, 2) = 4/3, F(2, 3) = 9/4, et F(4, 5) = 6875/24.

Soit G(N, M) = ∑_{n=2}^N ∑_{m=2}^M F(n, m). Par exemple, G(3, 3) = 137/12 et G(4, 5) = 6277/12. On vous donne aussi que G(6, 6) ≈ 1.681521567954e4 en notation scientifique avec 12 chiffres significatifs après la virgule décimale.

**Objectif** : Trouver G(123, 1234567). Donner votre réponse arrondie à 10 places après la virgule décimale.

---

## Exemples
### Exemple 1
**Entrée** : `n = 2, m = 2`
**Sortie** : `0.5`
**Explication** : F(2, 2) = 1/2.

---

## Approche de Résolution

### Stratégie
1. Modéliser le processus comme une chaîne de Markov.
2. Les états sont les distributions possibles des m balles parmi n bols.
3. Calculer les probabilités de transition entre états.
4. Utiliser les équations de récurrence pour calculer F(n, m).
5. Pour chaque état, calculer l'espérance du nombre d'étapes jusqu'à l'absorption.

### Observations
- C'est un problème de chaîne de Markov avec des états d'absorption.
- On peut utiliser la programmation dynamique pour calculer F(n, m).
- Pour de grandes valeurs, on doit optimiser considérablement.

### Complexité
- **Temporelle** : O(n^m × m) pour calculer F(n, m).
- **Spatiale** : O(n^m) pour stocker les états.
