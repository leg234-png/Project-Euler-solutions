# Somme de Pièces

## Description du Problème
Au Royaume-Uni, la monnaie est composée de livres (£) et de pence (p). Il y a huit pièces en circulation générale :

1p, 2p, 5p, 10p, 20p, 50p, £1 (100p), et £2 (200p).

Il est possible de faire £2 de la manière suivante :

1×£1 + 1×50p + 2×20p + 1×5p + 1×2p + 3×1p

**Objectif** : De combien de façons différentes peut-on faire £2 en utilisant n'importe quel nombre de pièces ?

---

## Exemples
### Exemple 1
**Entrée** : Montant = 5p
**Sortie** : `4`
**Explication** : 5p peut être fait de 4 façons : 5×1p, 3×1p+1×2p, 1×1p+2×2p, 1×5p.

---

## Approche de Résolution

### Stratégie
Utiliser la programmation dynamique pour résoudre le problème de somme de sous-ensembles :
1. Créer un tableau `ways[i][j]` représentant le nombre de façons d'utiliser les i premières pièces pour former la somme j.
2. Pour chaque pièce, mettre à jour le tableau en considérant les façons d'utiliser cette pièce.
3. Retourner `ways[n][TOTAL]`.

### Observations
- On peut optimiser l'espace en utilisant un tableau unidimensionnel.
- Pour chaque pièce de valeur `coin`, on met à jour : `ways[j] += ways[j - coin]` si `j >= coin`.
- L'ordre des pièces n'a pas d'importance, mais les valeurs doivent être uniques.

### Complexité
- **Temporelle** : O(n × m), où `n` est le nombre de pièces et `m` est le montant total.
- **Spatiale** : O(m), pour le tableau de programmation dynamique.

---

## Solution en C++

```cpp
#include <iostream>
#include <vector>

int main() {
    const int TOTAL = 200;
    std::vector<int> coins = {1, 2, 5, 10, 20, 50, 100, 200};
    
    std::vector<int> ways(TOTAL + 1, 0);
    ways[0] = 1;
    
    for (int coin : coins) {
        for (int j = coin; j <= TOTAL; ++j) {
            ways[j] += ways[j - coin];
        }
    }
    
    std::cout << ways[TOTAL] << std::endl;
    return 0;
}
```
