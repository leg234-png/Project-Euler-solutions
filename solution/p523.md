# Premier tri I

## Description du Problème
Considérons l'algorithme suivant pour trier une liste de nombres :

1. Si la liste est vide, arrêter.
2. Sinon, parcourir la liste de gauche à droite. Dès qu'on trouve deux éléments consécutifs dans le mauvais ordre :
   a. Déplacer le plus petit des deux à l'avant de la liste.
   b. Revenir à l'étape 1.

Soit F(L) le nombre de fois que l'étape 2a est exécutée pour trier la liste L. Par exemple, F({4,1,3,2}) = 5.

Soit E(n) la valeur attendue de F(L) sur toutes les permutations L des entiers {1,2,...,n}.

On vous donne que E(2) = 0.5, E(3) = 1.5, E(4) = 3.25, E(5) = 6.25, E(6) = 11.416666666666666, E(7) = 20.416666666666668, E(8) = 36.291666666666664, E(9) = 64.625.

**Objectif** : Trouver E(30). Donner votre réponse arrondie à 2 décimales après la virgule.

---

## Exemples
### Exemple 1
**Entrée** : `n = 2`
**Sortie** : `0.5`
**Explication** : E(2) = 0.5.

---

## Approche de Résolution

### Stratégie
1. Observer les valeurs de E(n) et les simplifier en fractions.
2. E(2) = 1/2, E(3) = 3/2, E(4) = 13/4, E(5) = 25/4, E(6) = 137/12, etc.
3. Rechercher dans OEIS : A330718 (numérateurs) et A330719 (dénominateurs).
4. La formule est : E(n) = ∑ₖ₌₁ⁿ (2^(k-1) - 1)/k.
5. Calculer cette somme pour n = 30.

### Observations
- Il existe une formule directe pour E(n).
- On peut calculer efficacement sans générer toutes les permutations.
- La formule utilise une série harmonique modifiée.

### Complexité
- **Temporelle** : O(n) pour calculer la somme.
- **Spatiale** : O(1).

---

## Solution en C++

```cpp
#include <iostream>
#include <iomanip>
#include <cmath>

using namespace std;

double compute(int n) {
    double total = 0.0;
    for (int k = 1; k <= n; k++) {
        total += (pow(2, k - 1) - 1.0) / k;
    }
    return total;
}

int main() {
    cout << fixed << setprecision(2) << compute(30) << endl;
    return 0;
}
```
