# Traînées de divisibilité

## Description du Problème
Soit streak(n) = k le plus grand entier positif tel que n+k-1 est divisible par k, mais n+k n'est pas divisible par k+1.

Par exemple, 120 est divisible par 1, 121 n'est PAS divisible par 2, donc streak(120) = 1.

Soit P(s, N) le nombre d'entiers n, 1 < n < N, pour lesquels streak(n) = s.

**Objectif** : Trouver ∑_{i=1}^{31} P(i, 4^i).

---

## Exemples
### Exemple 1
**Entrée** : `n = 120`
**Sortie** : `1`
**Explication** : streak(120) = 1.

---

## Approche de Résolution

### Stratégie
1. Observer que streak(n) = k signifie que k|n, (k+1)|(n+1), ..., (2k-1)|(n+k-1), mais (2k) ne divise pas (n+k).
2. Par le théorème des restes chinois, cela se réduit à : n ≡ 0 (mod lcm(1,2,...,k)) et n ≢ -k (mod lcm(1,2,...,k,k+1)).
3. P(s, N) = ⌊(N-2)/lcm(1,2,...,s)⌋ - ⌊(N-2)/lcm(1,2,...,s,s+1)⌋.
4. Calculer les LCM et appliquer la formule.

### Observations
- Le théorème des restes chinois permet de résoudre le système de congruences.
- On peut calculer efficacement les LCM.
- La formule permet de calculer P(s, N) sans itérer sur tous les nombres.

### Complexité
- **Temporelle** : O(s × log s) pour calculer chaque LCM.
- **Spatiale** : O(1).

---

## Solution en C++

```cpp
#include <iostream>
#include <vector>
#include <cmath>

using namespace std;

long long lcm(const vector<int>& numbers) {
    long long result = 1;
    for (int num : numbers) {
        result = result * num / __gcd(result, (long long)num);
    }
    return result;
}

long long P(int s, long long N) {
    vector<int> nums1;
    for (int i = 1; i <= s; i++) {
        nums1.push_back(i);
    }
    long long lcm1 = lcm(nums1);
    
    vector<int> nums2 = nums1;
    nums2.push_back(s + 1);
    long long lcm2 = lcm(nums2);
    
    return (N - 2) / lcm1 - (N - 2) / lcm2;
}

long long compute() {
    long long total = 0;
    for (int i = 1; i <= 31; i++) {
        long long N = pow(4, i);
        total += P(i, N);
    }
    return total;
}

int main() {
    cout << compute() << endl;
    return 0;
}
```
