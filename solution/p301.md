# Nim

## Description du Problème
Le jeu de Nim est joué avec des tas de pierres. À chaque tour, un joueur peut retirer n'importe quel nombre de pierres d'un seul tas.

Dans ce problème, on considère une variante avec trois tas de tailles `n`, `2n`, et `3n`.

**Objectif** : Combien de valeurs de `n` inférieures à `2^30` sont des positions perdantes (où le joueur qui commence perd si les deux joueurs jouent de manière optimale) ?

---

## Exemples
### Exemple 1
**Entrée** : `n = 1`
**Sortie** : Position gagnante ou perdante
**Explication** : Pour n=1, les tas sont {1, 2, 3}. 1 XOR 2 XOR 3 = 0, donc position perdante.

---

## Approche de Résolution

### Stratégie
1. Dans le jeu de Nim, une position est perdante si et seulement si le XOR de toutes les tailles de tas est 0.
2. On cherche `n` tel que `n XOR 2n XOR 3n = 0`.
3. Simplification algébrique :
   - `n XOR 2n XOR 3n = 0` si et seulement si `n & 2n = 0`.
   - Cela signifie que la représentation binaire de `n` n'a pas de bits consécutifs consécutifs.
4. Compter les nombres binaires de longueur 30 sans bits consécutifs :
   - C'est la suite de Fibonacci : `F(32)`.

### Observations
- Un nombre n'a pas de bits consécutifs si et seulement si `n & (n << 1) = 0`.
- Le nombre de chaînes binaires de longueur `i` sans bits consécutifs est `F(i+2)`.

### Complexité
- **Temporelle : O(log n)`.
- **Spatiale : O(1)**.

---

## Solution en C++

```cpp
#include <iostream>

using namespace std;

int main() {
    int a = 0, b = 1;
    for (int i = 0; i < 32; i++) {
        int c = a + b;
        a = b;
        b = c;
    }
    cout << a << endl;
    return 0;
}
```
