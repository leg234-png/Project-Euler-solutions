# Compositions à runs impairs

## Description du Problème
Soit F(n) le nombre de compositions de n où chaque run a une longueur impaire.

Par exemple, F(5) = 10 :
5, 4+1, 3+2, 2+3, 2+1+2, 2+1+1+1, 1+4, 1+3+1, 1+1+1+2, 1+1+1+1+1.

**Objectif** : Trouver F(10⁵). Donner votre réponse modulo 1111124111.

---

## Exemples
### Exemple 1
**Entrée** : `n = 5`
**Sortie** : `10`
**Explication** : F(5) = 10.

---

## Approche de Résolution

### Stratégie
1. Une composition de n est une façon d'écrire n comme somme d'entiers positifs.
2. Chaque run doit avoir une longueur impaire.
3. Utiliser la programmation dynamique : F(n) = nombre de compositions où tous les termes sont impairs.
4. F(n) = F(n-1) + F(n-3) + F(n-5) + ... (sommer sur tous les nombres impairs ≤ n).
5. Utiliser la mémorisation pour calculer efficacement.

### Observations
- Il y a une relation de récurrence claire pour F(n).
- On peut calculer F(n) de manière incrémentale.
- La mémorisation est essentielle pour les grandes valeurs.

### Complexité
- **Temporelle** : O(n²) pour calculer F(n) avec la récurrence.
- **Spatiale** : O(n) pour la table DP.

---

## Solution en C++

```cpp
#include <iostream>
#include <vector>

using namespace std;

long long mod = 1111124111;

long long compute(int n, long long mod) {
    vector<long long> F(n + 1, 0);
    F[0] = 1;
    
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= i; j += 2) { // Seulement les nombres impairs
            F[i] = (F[i] + F[i - j]) % mod;
        }
    }
    
    return F[n];
}

int main() {
    cout << compute(100000, mod) << endl;
    return 0;
}
```
