# Su Doku

## Description du Problème
Su Doku (signification japonaise : place de nombre) est le nom donné à un concept de puzzle populaire. Son origine n'est pas claire, mais le crédit doit être attribué à Leonhard Euler qui a inventé une idée de puzzle similaire, et beaucoup plus difficile, appelée Carrés Latins. L'objectif des puzzles Su Doku, cependant, est de remplacer les cases vides (ou zéros) dans une grille de 9 par 9 de telle sorte que chaque ligne, colonne et boîte de 3 par 3 contienne chacun des chiffres 1 à 9.

Un puzzle Su Doku bien construit a une solution unique et peut être résolu par la logique, bien qu'il puisse être nécessaire d'employer des méthodes de "deviner et tester" afin d'éliminer les options.

Le fichier texte de 6K, sudoku.txt, contient cinquante puzzles Su Doku différents de difficulté variable, mais tous avec des solutions uniques.

**Objectif** : En résolvant les cinquante puzzles, trouver la somme des nombres à 3 chiffres trouvés dans le coin supérieur gauche de chaque grille de solution ; par exemple, 483 est le nombre à 3 chiffres trouvé dans le coin supérieur gauche de la grille de solution ci-dessus.

## Exemples
### Exemple 1
**Entrée** : Une grille Su Doku partiellement remplie
**Sortie** : `483` (les trois premiers chiffres de la solution)
**Explication** : La grille est résolue et les trois premiers chiffres sont 483.

---

## Approche de Résolution

### Stratégie
1. Lire les 50 puzzles depuis le fichier.
2. Pour chaque puzzle, utiliser un algorithme de backtracking pour le résoudre.
3. Vérifier les contraintes (ligne, colonne, boîte 3×3) avant de placer un chiffre.
4. Extraire les trois premiers chiffres de chaque solution et les additionner.

### Observations
- Le backtracking est efficace pour résoudre les puzzles Su Doku.
- On doit vérifier trois contraintes pour chaque placement :
  - Le chiffre n'apparaît pas dans la ligne.
  - Le chiffre n'apparaît pas dans la colonne.
  - Le chiffre n'apparaît pas dans la boîte 3×3.
- On parcourt la grille de gauche à droite, de haut en bas.

### Complexité
- **Temporelle** : O(9^(cases_vides)) dans le pire cas, mais généralement beaucoup plus rapide avec le backtracking.
- **Spatiale** : O(1) pour la grille, O(n) pour la récursion où n est le nombre de cases vides.

## Solution en C++

```cpp
#include <iostream>
#include <string>
#include <vector>

bool solveSudoku(std::vector<std::vector<char>>& board, int index) {
    if (index == 81) {
        return true;
    }
    
    int row = index / 9;
    int col = index % 9;
    
    if (board[row][col] != '0') {
        return solveSudoku(board, index + 1);
    }
    
    // Essayer chaque chiffre possible
    for (char digit = '1'; digit <= '9'; digit++) {
        // Vérifier la ligne
        bool valid = true;
        for (int c = 0; c < 9; c++) {
            if (board[row][c] == digit) {
                valid = false;
                break;
            }
        }
        if (!valid) continue;
        
        // Vérifier la colonne
        for (int r = 0; r < 9; r++) {
            if (board[r][col] == digit) {
                valid = false;
                break;
            }
        }
        if (!valid) continue;
        
        // Vérifier la boîte 3×3
        int boxRow = row / 3 * 3;
        int boxCol = col / 3 * 3;
        for (int r = boxRow; r < boxRow + 3; r++) {
            for (int c = boxCol; c < boxCol + 3; c++) {
                if (board[r][c] == digit) {
                    valid = false;
                    break;
                }
            }
            if (!valid) break;
        }
        if (!valid) continue;
        
        // Placer le chiffre et continuer
        board[row][col] = digit;
        if (solveSudoku(board, index + 1)) {
            return true;
        }
        // Backtrack
        board[row][col] = '0';
    }
    
    return false;
}

}
```
