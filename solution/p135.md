# Same Differences

## Description du Problème
Étant donné que les entiers positifs, x, y et z, sont des termes consécutifs d'une progression arithmétique, la plus petite valeur de l'entier positif, n, pour laquelle l'équation, x² - y² - z² = n, a exactement deux solutions est n = 27 :
34² - 27² - 20² = 12² - 9² - 6² = 27.

Il s'avère que n = 1155 est la plus petite valeur qui a exactement dix solutions.

**Objectif** : Combien de valeurs de n inférieures à un million ont exactement dix solutions distinctes ?

---

## Exemples
### Exemple 1
**Entrée** : `n = 27`
**Sortie** : `2`
**Explication** : Il y a 2 solutions : (34,27,20) et (12,9,6).

---

## Approche de Résolution

### Stratégie
1. Soit x = m, y = m - k, z = m - 2k (progression arithmétique).
2. Par algèbre : x² - y² - z² = (m - k)(5k - m).
3. Pour que les termes soient positifs et décroissants : m > 0, k > 0, 2k < m, et 5k > m.
4. Donc m/5 < k < m/2.
5. Pour chaque m et k valides, calculer n = (m - k)(5k - m) et compter les solutions.
6. Compter combien de n ont exactement 10 solutions.

### Observations
- On peut limiter la recherche : m < 2 × 10⁶ car (m - k)(5k - m) > m/2.
- On utilise un tableau pour compter le nombre de solutions pour chaque n.

### Complexité
- **Temporelle** : O(n²), où n ≈ 2 × 10⁶.
- **Spatiale** : O(n) pour stocker les compteurs.

---

## Solution en C++

```cpp
#include <iostream>
#include <vector>

const int LIMIT = 1000000;

int main() {
    std::vector<int> solutions(LIMIT, 0);
    
    for (int m = 1; m < LIMIT * 2; m++) {
        for (int k = m / 5 + 1; k * 2 < m; k++) {
            long long temp = (long long)(m - k) * (k * 5 - m);
            if (temp >= LIMIT) break;
            solutions[temp]++;
        }
    }
    
    int count = 0;
    for (int x : solutions) {
        if (x == 10) {
            count++;
        }
    }
    
    std::cout << count << std::endl;
    return 0;
}
```
